What We Discovered Building Nitpick Senior

The biggest problem isn't finding bugs...

It's how reviewers communicate them.

The Problem

AI reviewers provide code suggestions

"Use pr.get_files() instead of repo.compare()"

What Happens

Developer applies fix mechanically

Without understanding the root cause

The Whack-a-Mole Problem

🔨

Same bug, different line
3 review cycles

Review 1: "get_changed_files() won't work for forks"
Fix: Changed to pr.get_files()
Review 2: "get_file_content() won't work for forks"
Fix: Same issue, different method
Review 3: "Context collector won't work for forks"
Fix: Still the same root cause!
Root Cause: self.repo always points to base repository, so ALL methods using it fail for fork content.

Suggestion-Driven vs Problem-Driven

Suggestion-Driven (Bad)

"This will fail for forks.

Use pr.get_files() instead of repo.compare()"
  • Developer copy-pastes fix
  • Doesn't understand WHY
  • Misses other affected methods
  • Multiple review cycles

Problem-Driven (Good)

"This will fail for forks.

Why: self.repo always points to the base repository. Any method using it cannot access fork content."
  • Developer understands root cause
  • Asks: "Where else is this pattern?"
  • Fixes ALL affected methods
  • Single review cycle

AI Reviewers Can Be Confidently Wrong

Flagged 3 Times as ERROR:

"truncate_diff(diff_content, max_tokens) passes two arguments,
but the function only accepts one. This will raise TypeError."

Reality:

def truncate_diff(diff_content: str, max_tokens: int = MAX_DIFF_TOKENS) -> str:

The function DOES accept two arguments. The LLM assumed based on common patterns without verifying.

Lesson: AI reviewers make assumptions. They don't actually read and verify the full codebase.

Key Learnings

1. Don't Suggest Fixes

AI reviewers should identify WHAT and WHY, not HOW. Code suggestions create feedback loops.

2. Root Cause Matters

Explaining the underlying issue leads to complete fixes. Surface-level feedback leads to whack-a-mole.

3. Context Has Limits

Repository context helps, but AI still makes assumptions. It doesn't truly "understand" your codebase.

4. DIY is Viable

You can build 80% of paid tool functionality at 3% of the cost. The gap is shrinking.
